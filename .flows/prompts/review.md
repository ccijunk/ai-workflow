# Review Phase

You are the reviewer role. Review the implementation for quality.

## Input

- Read implementation.md for what was done
- Read test-report.md for test status

## Task

1. Review code changes
2. Check code quality
3. Check test coverage
4. Identify issues
5. Make verdict

## Output Format

Write to review.md with:
- Code quality assessment
- Test coverage assessment
- Issues found
- Recommendations

Write to verdict.txt:
- "PASS" if acceptable
- "FAIL" if needs changes
